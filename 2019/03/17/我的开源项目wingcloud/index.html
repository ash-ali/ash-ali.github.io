我的开源项目wingcloud

发表于 2019-03-17 | 分类于 随笔生活 | | 阅读次数:

【wingcloud传送门】：https://github.com/ash-ali/wingcloud 创作不易，欢迎star

今天2019-03-17，我的第一个开源项目终于公开了，今天我就聊聊在做开源项目的一些心得体会。

wingcloud这个项目是我在2018年5月份的时候突然想做的，因为当时觉得自己学了东西没有地方去使用，久而久之，许多知识又渐渐忘记了。于是那个时候就想自己做一个开源项目，就算帮不了别人，也算自己对知识的一个体系构建过程。起初 wingcloud 的设计思想并不是现在的这样，刚开始我想以搜索为主线，数据以批处理为主的系统，架构还是用微服务。
之前的设计思想也持续了很久，并搭建了基本的框架出来。直到大概2018年的9月份，我去了云栖大会，阴差阳错的走进了一个会场(Flink会场)，当时我就被这个技术吸引了，被它吸引的点主要是它是为了流计算而生，并且在批流统一上会很友好(虽然wingcloud现在还没有实现批流统一)，当时我就觉得可以考虑使用Flink对数据大屏的实时数据进行计算处理，
于是就开始看Flink官方文档，由于Flink的中文文档非常少，所以在学习Flink的时间上花费了很多，也在学习之余将知识整理到了仓库中：https://github.com/ash-ali/MyFlink-doc，然后继续开始了我的wingcloud之旅。

回到正题，聊聊wingcloud吧，wingcloud是我用SpringCloud来搭建的一个平台，之所以选择SpringCloud是因为SpringCloud社区在最近几年非常活跃，这样使用它也可以非常放心。然后在计算方面使用的是Flink，流计算处理引擎，目前许多大厂都在使用，而且阿里的双十一计算引擎就是基于Flink开发了自己的Blink。

在写自己的第一个开源项目其实发现并不简单，因为项目并不是自己使用就好，需要别人能看的舒服，所以我在其中也学到了许多规范，也参考了许多规范，即使现在还有许多不足。我也在wingcloud的README中列了v1.0.1版本会实现的功能。

如今笔者已经大三了马上就要准备实习了，可能没有像在学校里这么多时间去做这些事情，但还是会坚持不断的对wingcloud进行改进，重构。也非常希望有共同爱好的小伙伴提出意见，我一定会在第一时间看到的哈。

最后就放上项目的部分截图吧，可能图片被压缩或者不清楚，请见谅。
